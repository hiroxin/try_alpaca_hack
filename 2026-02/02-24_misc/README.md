# 2026/02/24 問題

https://alpacahack.com/daily/challenges/hit-and-hit

## 問題文

```
Hit! Hit! Hit! AAAAAAAAH LET'S GOOOOOOOO!!!

Hint: We recommend solving hit-and-miss first.
```

## 考察

コードを見ると入力した正規表現が一致していようがいなかろうが、`Hit!!!` が出力される。そのため、 Time-based Blind Injection（時間差を利用した推測攻撃）で成否を判定するしかない。

```
while pattern := input("regex> "):
    re.match(pattern, FLAG)
    print("Hit!!!")
```

成否に応じて応答時間差を生じさせるために ReDoS (Regular Expression Denial of Service：正規表現サービス拒否攻撃) を利用する。

ReDoS とは正規表現エンジンの「後戻り（バックトラック）」の仕様を悪用し、サーバーの計算量を爆発させてフリーズさせる攻撃のこと。 \
複雑な正規表現（特に繰り返しのネスト）に対して、文字列の最後までマッチしなかった場合、エンジンは「別の区切り方ならマッチするかもしれない」と、ありとあらゆる組み合わせの総当たり計算を始める。この計算量は指数関数的に増大するため、意図的にサーバーの応答を遅延させることができる。

このReDoSの性質とTime-based Blind攻撃を掛け合わせ、以下のロジックで1文字ずつフラグを特定する。

1. もし推測した文字が当たっていたら、ReDoSを実行してサーバーを重くする
2. もし外れていたら処理を終了する

このペイロードを a から順にサーバーに送信し、応答時間が「ネットワークの往復時間（約0.15秒）」であればハズレ、「閾値（約0.5秒以上）」を超えればアタリと判断する。

相応の試行回数が必要になるため、手打ちではなくスクリプトで自動化する。

### 実装方針

ペイロードは以下の通り。

`^(?=Alpaca\{` + 任意の文字列 + `)` + `(?:.|[\s\S]|[\w\W]|[\d\D]|[^\n]|[^\t]|[^\r]|[^\f]){8}!`

- `^(?=Alpaca\{<<任意の文字>>)`
  - 「肯定先読み」と呼ばれる構文。「先頭が `Alpaca{` + 任意の文字列 + で始まっているか」をチェックし、合っていれば以降のペイロードの確認に進み、間違っていればその瞬間にマッチングを終了させる。

- `(?:.|[\s\S]|[\w\W]|[\d\D]|[^\n]|[^\t]|[^\r]|[^\f]){8}!`
  - 先のマッチが一致した場合にのみ実行される。中には「絶対にどんな文字にもマッチする8個の条件」が並んでおり、それを8回繰り返すよう指示している。最後にフラグ末尾に絶対に置かれない `!` があるため必ずマッチングが失敗し、エンジンは8の8乗（約1,677万回）の組み合わせを強制的に総当たりさせられる。これによって 0.5秒以上の遅延を発生させる。
  - 正規表現内訳
    - `(?: )` : キャプチャなしグループ。カッコを一つのパーツとしてまとめる
      - `()` と異なり、マッチした文字列をメモリに記憶させない
    - `.` : 改行以外の任意の一文字
    - `|` : OR 条件
    - `[\s\S]` : 任意の空白文字と任意の非空白文字
    - `[\w\W]` : 任意の英数字と任意の非英数字
    - `[\d\D]` : 任意の数字と任意の非数字
    - `[^\n]` : 改行文字以外
    - `[^\t]` : タブ文字以外
    - `[^\r]` : 復帰文字以外
    - `[^\f]` : 改ページ文字以外
    - `{8}` : 直前の条件を 8 回繰り返す

流れを改めて整理する。

1. 前者のマッチングで現在の試行までのフラグが正しいかを確認する。
2. 前者が正しければ、応答時間を遅延させるため後者のマッチングを行う。前者のマッチングが失敗した場合はそのまま処理は終了。次の試行に進む。
3. 後者のマッチングを行う。

- 3-a. まず `.` だけでマッチングが行われ、9文字目の `!` でマッチングが失敗する。
- 3-b. バックトラックが発生し、末尾だけ `[\s\S]` に変えてマッチングを行う。同じく 9 文字目で失敗する。
- 3-c. これを 8^8 回繰り返す。

4. その間に応答時間が 0.5 秒を超えるため、スクリプトは前者のペイロードがフラグと一致していると判断する。
